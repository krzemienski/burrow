Configuration

Burrow's Settings window has five tabs. This reference explains every control, where its value is stored, and what happens when you change it.

General tab

Launch at login

Toggle registers or deregisters Burrow with SMAppService.mainApp. When enabled, macOS starts Burrow as part of the login sequence — before the Dock appears. The tunnel is live within seconds of desktop load. Stored as a boolean in PreferencesStore (UserDefaults key launchAtLogin). Registration fails silently if the app is not in /Applications; Burrow detects this and shows a prompt to move the app.

Notifications

Enables or disables UNUserNotificationCenter banners. When first toggled on, Burrow requests notification authorization. The following events trigger a notification: token revoked externally, three or more tunnel failures within a five-minute window, cloudflared binary not found at startup. Silent reconnects (network change, sleep/wake) never trigger notifications.

Cloudflare tab

API token

Shows the last four characters of the stored token for identification. To replace it, click Change — this opens a secure input sheet, validates the new token against GET /user/tokens/verify, checks all four required scopes, and writes to Keychain only on success. The old token is overwritten atomically. If the new token fails scope validation, the existing token is preserved and an error sheet lists the missing permissions.

Account

The Cloudflare account ID used for all tunnel and zone operations. Set during the wizard and read-only after setup. To change accounts, run the tunnel uninstall flow (Advanced tab → Delete tunnel), which stops cloudflared, deletes the tunnel via API, removes the CNAME record, and clears Keychain. Then re-run the wizard.

Zone

The zone name (e.g. hack.ski) and zone ID. Read-only after setup. Changing zones requires uninstalling and re-provisioning the tunnel because the CNAME record is bound to a specific zone's DNS.

Tunnel tab

Tunnel name

The human-readable name passed to POST /accounts/{id}/cfd_tunnel during setup. Format: burrow-<subdomain> (e.g. burrow-m4). Read-only after creation. The wizard blocks creation if the name already exists in the account and offers to reuse the existing tunnel's run token instead.

Tunnel UUID

The UUID returned by Cloudflare at tunnel creation. This is the stable identifier that appears in the cfargotunnel.com CNAME target. Shown for reference only; not editable.

Ingress service

Fixed to ssh://localhost:22 in v1.0. The ingress array stored in Cloudflare always contains two rules: the hostname rule pointing to ssh://localhost:22 and a mandatory catch-all http_status:404. Cloudflare rejects any ingress array whose last rule has a hostname — the catch-all is required by the API.

Reconnect behavior

Controls the backoff when cloudflared exits unexpectedly. Default: exponential starting at 2 seconds, doubling each attempt, capped at 60 seconds. After three failures within five minutes, Burrow posts a persistent notification and pauses retries until you click Retry in the menu.

DNS tab

Hostname

The full FQDN of the CNAME record (e.g. m4.hack.ski). Provisioned during setup. This is the address you use in your SSH command and in ~/.ssh/config. Read-only after setup.

CNAME target

Always <tunnel_uuid>.cfargotunnel.com with proxied: true. The proxied flag is mandatory — a non-proxied CNAME to cfargotunnel.com returns NXDOMAIN because Cloudflare does not expose an underlying IP without proxying enabled.

DNS record ID

Cloudflare's internal identifier for the CNAME record, returned by POST /zones/{zone_id}/dns_records. Stored in PreferencesStore so Burrow can update or delete the record without an extra lookup round-trip on every operation.

Advanced tab

cloudflared path

The filesystem path to the cloudflared binary. BinaryLocator searches /opt/homebrew/bin, /usr/local/bin, and /usr/bin in that order. You can override with a custom path — useful if cloudflared is installed by Nix or MacPorts at a non-standard location. The path is validated with cloudflared --version before saving; the semver is parsed from the output using the pattern version (\d+\.\d+\.\d+).

Log level

Sets the minimum OSLog level for the com.krzemienski.burrow subsystem. Options: info (default), debug, error. Debug mode logs every API request URL and HTTP status code. Raw tokens are always scrubbed — the pattern --token \S+ is replaced with --token [REDACTED] before any string reaches OSLog, regardless of log level.

cloudflared invocation

Burrow uses cloud-managed mode by default:

cloudflared tunnel run --token <run_token_base64>

The run token is retrieved from Keychain at startup, not stored in any file. If you need the local-managed override (Advanced → Expert mode), Burrow instead uses:

cloudflared tunnel --config /path/to/config.yml run <tunnel_uuid>

Local mode is not recommended — it requires a YAML config file on disk and disables cloud-side ingress management via the API.

Delete tunnel

Uninstall flow: stops cloudflared (SIGTERM, 5-second grace, then SIGKILL), waits 10 seconds for Cloudflare to register zero active connections, calls DELETE /accounts/{id}/cfd_tunnel/{tunnel_id} with up to three retries, then deletes the CNAME record. Clears both Keychain items. Resets PreferencesStore to factory defaults. On next launch the first-run wizard opens again.

Delete is permanent. The tunnel UUID is gone. The DNS record is deleted. SSH connections using this hostname will fail immediately. There is no undo. Update any ~/.ssh/config entries on client machines before proceeding.
