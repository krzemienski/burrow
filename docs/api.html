Cloudflare API Reference

Every Cloudflare API v4 endpoint Burrow calls, with the exact request shape and expected response. Use these with curl to debug token issues or verify state outside the app.

Authentication

All requests use Bearer token authentication. Base URL: https://api.cloudflare.com/client/v4. Every request must include:

Authorization: Bearer <your_api_token>
Content-Type: application/json

Burrow uses a single URLSession for all API calls and applies exponential backoff on HTTP 429 responses. Cloudflare rate limit: 1200 requests per 5-minute window per account across all clients sharing the token.

Token verification

Called immediately after token entry in the wizard and on every app launch to confirm the token is active.

GET /user/tokens/verify

200 OK
{
  "result": { "id": "abc123", "status": "active" },
  "success": true,
  "errors": [],
  "messages": []
}

401 Unauthorized  ← token invalid or revoked

Burrow parses the status field. Any value other than "active" triggers the red-dot state and a re-authorize notification. After token entry, Burrow also checks that all four required permission scopes are present in the token's policy list.

Account discovery

Fetches all accounts the token can access. Used in wizard step 3 to populate the account picker. If only one account is returned, the picker is skipped.

GET /accounts

200 OK
{
  "result": [
    { "id": "<account_id>", "name": "Personal Account" }
  ],
  "success": true
}

Zone discovery

Lists all zones accessible to the token. Paginated at 50 per page. Used in wizard step 4 to populate the zone picker.

GET /zones?per_page=50

200 OK
{
  "result": [
    {
      "id": "<zone_id>",
      "name": "hack.ski",
      "status": "active",
      "account": { "id": "<account_id>" }
    }
  ],
  "success": true
}

Create tunnel

Creates a named, cloud-managed tunnel. Called once during wizard step 7. The config_src: "cloudflare" field is mandatory — omitting it defaults to "local" mode, which requires a YAML config file on disk and prevents the ingress PUT API from working.

POST /accounts/{account_id}/cfd_tunnel
{
  "name": "burrow-m4",
  "config_src": "cloudflare"
}

200 OK
{
  "result": {
    "id": "<tunnel_uuid>",
    "name": "burrow-m4",
    "token": "<run_token_base64>"
  },
  "success": true
}

The token in the response is the run token passed to cloudflared tunnel run --token. Burrow stores it in Keychain immediately and never logs it.

Get run token

Re-fetches the run token for an existing tunnel. Used during recovery if the Keychain item is missing or corrupt.

GET /accounts/{account_id}/cfd_tunnel/{tunnel_id}/token

200 OK
{
  "result": "<run_token_base64>",
  "success": true
}

Set ingress configuration

Pushes ingress rules to Cloudflare's edge. Called during wizard step 7 after tunnel creation. The catch-all rule with no hostname is required as the last entry — the API rejects any ingress array whose final rule has a hostname field.

PUT /accounts/{account_id}/cfd_tunnel/{tunnel_id}/configurations
{
  "config": {
    "ingress": [
      {
        "hostname": "m4.hack.ski",
        "service": "ssh://localhost:22"
      },
      {
        "service": "http_status:404"
      }
    ]
  }
}

200 OK
{ "result": { ... }, "success": true }

List tunnels

Lists all non-deleted tunnels in the account. Used during wizard step 5 to detect name conflicts and during the recovery flow to find an existing tunnel by name.

GET /accounts/{account_id}/cfd_tunnel?is_deleted=false

200 OK
{
  "result": [
    { "id": "<uuid>", "name": "burrow-m4", "status": "healthy" }
  ],
  "success": true
}

Delete tunnel

Permanently deletes a tunnel. The tunnel must have zero active connections. Burrow stops cloudflared, waits 10 seconds, then retries this call up to three times before reporting failure.

DELETE /accounts/{account_id}/cfd_tunnel/{tunnel_id}

200 OK
{ "result": { "id": "<tunnel_uuid>" }, "success": true }

409 Conflict  ← tunnel still has active connections, retry after delay

DNS record CRUD

Burrow creates, checks, and deletes one CNAME record. The record must use proxied: true — a non-proxied CNAME to cfargotunnel.com returns NXDOMAIN because Cloudflare does not expose an underlying IP without the proxy layer.

// Check for existing record before creating
GET /zones/{zone_id}/dns_records?name=m4.hack.ski&type=CNAME

// Create record
POST /zones/{zone_id}/dns_records
{
  "type": "CNAME",
  "name": "m4.hack.ski",
  "content": "<tunnel_uuid>.cfargotunnel.com",
  "proxied": true,
  "ttl": 1
}

200 OK
{
  "result": { "id": "<record_id>", "name": "m4.hack.ski" },
  "success": true
}

// Delete record (uninstall flow)
DELETE /zones/{zone_id}/dns_records/{record_id}

ttl: 1 means automatic TTL managed by Cloudflare. The record ID returned by POST is stored in PreferencesStore for future update and delete operations without an extra lookup.

Manual curl examples

To debug a token issue outside Burrow:

export CF_TOKEN="your_token_here"
export ACCOUNT_ID="your_account_id"

# Verify token is active
curl -s -H "Authorization: Bearer $CF_TOKEN" \
  https://api.cloudflare.com/client/v4/user/tokens/verify \
  | python3 -m json.tool

# List non-deleted tunnels
curl -s -H "Authorization: Bearer $CF_TOKEN" \
  "https://api.cloudflare.com/client/v4/accounts/$ACCOUNT_ID/cfd_tunnel?is_deleted=false" \
  | python3 -m json.tool
Rate limits. Cloudflare enforces 1200 requests per 5 minutes per account across all clients using the same token. Manual curl tests while Burrow is running count against the same limit.
