Installation

From a notarized DMG to an active SSH tunnel in under five minutes.

Prerequisites

Before opening the Burrow DMG, verify three things are in place:

  1. macOS 13 Ventura or later. Burrow uses MenuBarExtra and SMAppService, both introduced in macOS 13.
  2. A Cloudflare account with an active zone. Your domain must be delegated to Cloudflare's nameservers and the zone status must be Active at dash.cloudflare.com.
  3. cloudflared binary. Install with Homebrew: brew install cloudflared. Verify with cloudflared --version — Burrow expects 2024.x.x or later and parses the semver from the version string.

Step 1 — Create an API token

Burrow requires a Cloudflare API token with exactly four permission scopes. The token is stored in your macOS Keychain (kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly) and never written to disk, never logged, never stored in UserDefaults.

Go to dash.cloudflare.com/profile/api-tokens, click Create Token, and choose Custom token. Add these four permissions:

  • Account → Cloudflare Tunnel → Edit
  • Zone → DNS → Edit
  • Zone → Zone → Read
  • Account → Account Settings → Read

Under Zone Resources, select the specific zone Burrow will manage. Set a TTL matching your security policy — 90 days is reasonable for a personal machine.

Token security. Copy the token immediately after creation — Cloudflare shows it only once. Never paste it into a terminal command that lands in shell history. Burrow's wizard accepts the token in a secure text field that does not echo to logs.

Step 2 — Install Burrow

Download the latest notarized DMG from the Releases page. Open the DMG and drag Burrow.app to /Applications.

Must be in /Applications. Burrow registers a launch-at-login item via SMAppService.mainApp.register(). macOS requires the app bundle to be in /Applications for this API to succeed. Launching from Downloads or directly from the DMG will cause the launch-at-login toggle to fail silently.

Launch Burrow from /Applications. No Dock icon appears — this is intentional (LSUIElement = YES in Info.plist). A gray network icon appears in your menu bar.

Step 3 — Run the first-run wizard

On first launch the setup wizard opens automatically and walks through seven steps:

  1. Welcome. Summary of what Burrow will create on your Cloudflare account.
  2. API token. Paste your token. Burrow immediately calls GET /user/tokens/verify and checks all four required scopes. A scope diff is shown if any are missing — the wizard will not advance until all four pass.
  3. Account. If the token has access to multiple accounts, select the one containing your zone.
  4. Zone. Burrow lists all zones accessible to the token. Select the zone (e.g. hack.ski).
  5. Subdomain. Enter a hostname prefix (e.g. m4). Burrow checks for an existing CNAME at that FQDN and warns if one already exists.
  6. cloudflared check. Burrow runs cloudflared --version. If the binary is not found at any of the expected paths (/opt/homebrew/bin, /usr/local/bin, /usr/bin), this step shows the install command and waits for you to install and retry.
  7. Create tunnel. Burrow calls POST /accounts/{id}/cfd_tunnel with config_src: "cloudflare", PUTs the ingress config with ssh://localhost:22 and the mandatory catch-all http_status:404, creates the CNAME DNS record with proxied: true, stores the run token in Keychain, then starts cloudflared tunnel run --token <run_token>.

When the wizard completes, the menu bar icon turns orange and the tunnel is live. Click the icon and select Copy SSH command to copy ssh user@m4.hack.ski to the clipboard.

Step 4 — Configure your SSH client

Cloudflare Tunnel routes SSH through the Cloudflare edge. Your client needs cloudflared installed and the following block in ~/.ssh/config:

Host *.hack.ski
  ProxyCommand cloudflared access ssh --hostname %h
  StrictHostKeyChecking no

Replace hack.ski with your own zone. This block tells ssh to proxy all connections through cloudflared access, which handles the Cloudflare authentication handshake before the TCP connection reaches your Mac.

Step 5 — Verify from a different network

Switch to a mobile hotspot (to confirm NAT traversal works), then run:

ssh nick@m4.hack.ski

A successful connection confirms the tunnel, DNS record, ingress config, and SSH proxy are all working. Burrow's menu shows tunnel up with an uptime counter.

Step 6 — Enable launch at login

Open Burrow → Settings → General and toggle Launch at login. Burrow calls SMAppService.mainApp.register(). On the next reboot, Burrow starts automatically before any user interaction — the tunnel will be live within seconds of your Mac finishing its boot sequence.
