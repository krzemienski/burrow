Burrow Documentation

Burrow is a notarized macOS menu bar application that exposes your Mac's SSH daemon over a stable Cloudflare Tunnel — no port forwarding, no dynamic DNS, no configuration files.

What Burrow does

Burrow automates the full Cloudflare Tunnel setup that would otherwise require a terminal session, a YAML config file, a launchd plist, and manual DNS management. You paste one API token. Burrow creates a named tunnel, provisions a CNAME record (m4.hack.ski or any subdomain you choose), and starts cloudflared as a managed child process. The menu bar icon flips from gray to orange when the tunnel is live.

Because the tunnel uses Cloudflare's QUIC-based edge network for the outbound connection, SSH reaches your Mac even when it sits behind NAT, CGNAT, or a firewall that blocks all inbound TCP.

Architecture in one paragraph

Burrow is a single SwiftUI MenuBarExtra app (no Dock icon, LSUIElement = YES). It holds one CloudflareClient actor wrapping URLSession for all CF API v4 calls and one CloudflaredManager actor that owns the Process running cloudflared tunnel run --token <run_token>. Your API token and the tunnel run token are stored in the system Keychain under com.krzemienski.burrow — they never appear in logs, UserDefaults, or any file on disk.

Key properties

  • Stable hostname. The CNAME is created once and never changes. The tunnel UUID is the permanent address; cfargotunnel.com handles routing.
  • Keychain-only secrets. API token and run token live in kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly items. Never logged.
  • Sleep and wake aware. CloudflaredManager listens to system power notifications. Tunnel stops on sleep, restarts within 10 seconds on wake.
  • Network resilient. NWPathMonitor triggers reconnect on interface change. Exponential backoff caps at 60 seconds.
  • No sandbox for v1.0. Hardened Runtime is on; App Sandbox is off so Process.launch() can reach cloudflared in /opt/homebrew/bin.

Requirements

  • macOS 13 Ventura or later
  • A Cloudflare account with at least one active zone
  • cloudflared installed via brew install cloudflared
  • A Cloudflare API token with four specific permission scopes (see Installation → Create an API token)

Documentation map

Start with Installation if you are setting up Burrow for the first time. The wizard covers token creation, zone selection, subdomain choice, and tunnel provisioning end to end.

Once running, Configuration explains every setting in the Settings window. Cloudflare API documents every endpoint Burrow calls so you can verify permissions or debug token issues manually with curl. Troubleshooting covers the most common failures and how to read the unified system log.
